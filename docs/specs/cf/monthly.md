# 月次収支リスト(観測: 2026-07-11)

## エンドポイント

- `GET /cf/monthly`(HTML)。パラメータなし(表示期間は直近6ヶ月固定で観測)

## 構造

- `section#monthly-detail` 内の `table#monthly_list`(thead/tbody なし、`tr` 直下)
- ヘッダ行: 先頭は空 `th`(幅指定のみ、**閉じタグがなく後続の th を入れ子に抱き込む**)、
  以降 `th#js-th-<n>` に期間開始日ラベル
  (例 `2026/01/25〜`。締め日起点なので日付は 25 日等になる)
- 解析時は「th を子に持たない葉の th」だけを期間ラベルとして拾うこと
- データ行の `tr` クラスで行種別を判定する:
  - `in_sum`: 収入合計(`td.title`)
  - `in`: 収入カテゴリ(`td.item`)
  - `out_sum`: 支出合計(`td.title`)
  - `out`: 支出カテゴリ(`td.item`。食費・日用品・…・未分類の約20行)
  - `total`: 収支合計(`td.title`。負値あり: `-567,920円`)
- 金額セル: `td.number`(`id=js-in-total-<n>` 等。列順はヘッダと同順)
- ページ上部に推移グラフ用の div があるが、データは別途 JS 描画のため解析対象外

## 失敗の見分け方

- 未認証: `302 /users/sign_in`
- `table#monthly_list` 欠落: 仕様変更の可能性
